# m-official-notes

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态文件形式存放，不依赖于特定域名或网站系统。

## 项目简介

m-official-notes 是一个以 HTML 页面为单位的归档仓库。每个页面独立存在，可直接通过浏览器打开查看。仓库本身不包含后端服务或动态内容，仅提供结构化的页面集合。

## 目录说明

```
m-official-notes/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面引用的资源文件（图片、样式、脚本等）
└── README.md       # 本文件
```

- `pages/` 目录下每个子文件夹对应一个独立页面，文件夹名称即页面标识。
- `assets/` 目录按页面分类组织资源，避免文件冲突。

## 页面归档说明

- 每个页面均为独立完整的 HTML 文件，可脱离网络环境本地运行。
- 页面间无相互依赖关系，可单独修改、删除或新增。
- 归档内容遵循静态页面规范，不包含动态数据接口或外部资源链接。

## 维护说明

- 如需新增页面，请在 `pages/` 下创建新文件夹，并放入 index.html 及相关资源。
- 修改现有页面时，请确保不影响其他页面的正常访问。
- 删除页面时，请同步清理 `assets/` 中对应的冗余资源文件。
- 提交前建议本地预览，确认页面显示正常。

## 使用方式

1. 克隆本仓库到本地。
2. 在浏览器中直接打开 `pages/` 下任意页面的 index.html 文件。
3. 或通过任何静态文件服务器托管 `pages/` 目录进行访问。
